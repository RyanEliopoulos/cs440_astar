AStar

AStar! In this assignment, you'll implement the A Search algorithm. The stub code provides the framework for running A on a generic SearchProblem. Two SearchProblem classes are (partially) given: both work on simple 2D grids: the first on a grid represented by a list of lists, the second on a grid represented as pixels from an image.

Overview

  1. Download the stub astar.py.
  2. Look through the code. I've written some functionality, and the goal is for you to complete it.
  3. Note that there are 2 methods/functions that are unimplemented, these are where you should add your code.
  4. As before the code assumes four spaces of indentation. Set your editor appropriately, this is the default for many.

Notes on the Code

I've added ome descriptive doc-strings and comments. My hope is that these will help you understand the code better and write more idiomatic python. Here are a few items worth calling out:

heapq the heapq module implements heap operations on a list. It is worth reading about.

namedtuple named tuples create objects that are immutable (like tuples) but whose data can be accessed by name (instead of by an integer index as in a normal tuple). The syntax for creating a namedtuple is relatively strange so there is a comment to explain it.
